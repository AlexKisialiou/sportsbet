## Роли пользователей

| Роль | Описание |
|---|---|
| **Пользователь** | Делает ставки, видит таблицу и прогнозы других |
| **Администратор** | Управляет матчами, запускает Бендера, симулирует результаты |
| **Суперадмин** | Полный доступ: управление пользователями, сброс базы, смена темы, лог активности |

Суперадмин всегда имеет права администратора. Администратор не имеет прав суперадмина.

Суперадмин входит с паролем из переменной окружения `ADMIN_PASSWORD` (пароль не хранится в базе).

## Вход и профиль

### Вход (`/login`)

- Введи логин и пароль
- Боты (Бендер) не могут входить в систему
- После входа — редирект на главную страницу
- Лимит попыток: 15 в минуту, 60 в час (защита от брутфорса)

### Выход (`/logout`)

Очищает сессию и перенаправляет на страницу входа.

### Профиль (`/profile`)

**Аватар и никнейм:**
- Никнейм — отображаемое имя вместо логина. Если не задан, используется логин.
- Эмодзи-аватар: выбор из 24 вариантов (или инициал «Аа» по умолчанию)
- Цвет фона аватара: 12 цветов на выбор
- Превью аватара обновляется в реальном времени при выборе

**Смена пароля** (недоступна суперадмину):
- Нужно ввести текущий пароль, новый (минимум 3 символа) и подтверждение
- Суперадмин меняет свой пароль только через переменную окружения `ADMIN_PASSWORD`

## Главная страница (`/`)

Доступна только авторизованным пользователям. Состоит из трёх зон:

### Левая колонка — Очки

**Статус ставок:** зелёная плашка «Все ставки сделаны» или янтарная «N из N осталось» — показывает, сколько матчей ещё без прогноза у текущего пользователя.

**Турнирная таблица:** список всех участников с итоговыми очками и разбивкой по последним игровым дням. Подробнее — в разделе [Турнирная таблица](#турнирная-таблица).

**Ставки на ближайшие матчи:** таблица с отметками — кто уже сделал ставки на предстоящие матчи (зелёная галочка = все ставки сделаны).

### Правая колонка — Ставки

Карточки предстоящих матчей, отмеченных администратором как «для ставок». Для каждого матча:
- Названия и гербы команд
- Дата и время начала (UTC)
- Два поля ввода счёта (0–20) — для хозяев и гостей

Ставка сохраняется автоматически при изменении любого поля. Подробнее — в разделе [Как делать ставки](#как-делать-ставки).

### Нижняя зона — Предикты

Таблица завершённых матчей за последние 4 игровых дня (не более 10 матчей). Колонки: матч (с гербами команд), итоговый счёт, прогнозы каждого участника.

Цвета ячеек прогнозов:
- **Золотой** — угадал точный счёт (+3 очка)
- **Сине-зелёный** — угадал исход (победитель или ничья) (+1 очко)
- **Красный** — промах (0 очков)
- **Приглушённый** — ставки не было

---

## Как делать ставки

1. На главной странице в правой колонке найди карточку матча
2. Введи предполагаемый счёт в поля «хозяева» и «гости»
3. Ставка сохраняется автоматически — появится зелёная галочка «✓»
4. Ставку можно изменить в любой момент до начала матча (пока статус `scheduled`)

**Ограничения:**
- Ставки принимаются только на матчи, отмеченные администратором как «для ставок» (поле `featured`)
- Лимит запросов: 60 ставок в минуту (защита от спама)
- Матч должен иметь статус `scheduled`

## Система очков

| Результат | Очки |
|---|---|
| Точный счёт | **3** |
| Угадал исход (победа / ничья) | **1** |
| Промах | **0** |

**Примеры:**
- Реальный счёт 2:1, прогноз 2:1 → **3 очка** (точный счёт)
- Реальный счёт 2:1, прогноз 3:1 → **1 очко** (угадал победу хозяев)
- Реальный счёт 2:1, прогноз 1:1 → **0 очков** (промах — предсказал ничью)

Очки рассчитываются автоматически при обновлении статуса матча на `finished`.

## Турнирная таблица

Показывает всех участников (кроме ботов), отсортированных по сумме очков за все матчи.

**Колонки:**
- Место (позиция)
- Аватар и имя участника (с ролевым значком для админов)
- Итого очков
- Очки за каждый из последних 4 игровых дней:
  - `+N` зелёным — заработал N очков
  - `0` красным — ставка была, но промах
  - `—` серым — ставки не было

Игровой день = уникальная дата, в которую были завершённые матчи.

## Комментарии Бендера

Бендер — бот с характером, генерирует комментарии через Groq API (модель `llama-3.1-8b-instant`).

**Два типа комментариев:**

1. **Прогноз на матчи** (зелёная панель «📋 Прогноз»):
   - Генерируется при нажатии «Сохранить выбор» в админ-панели
   - Один прогноз на каждый «featured» матч
   - Бендер также делает свою ставку (сохраняется в таблице предиктов)

2. **Комментарий об очках** (золотая панель «📊 Бендер об очках»):
   - Генерируется после симуляции результатов
   - Комментирует турнирную таблицу и последний игровой день

Панели открываются кнопками в нижней части экрана. Закрываются кликом на затемнённый фон или повторным нажатием.

## Панель администратора (`/admin`)

Доступна пользователям с флагом `is_admin` или `is_superuser`.

### Загрузить матчи ЛЧ

Кнопка «Загрузить матчи ЛЧ» — делает запрос к football-data.org API и синхронизирует матчи Лиги Чемпионов. Показывает количество добавленных и обновлённых матчей.

Лимит: 5 запросов в минуту.

### Матчи для ставок

Чеклист всех запланированных матчей (UCL + АПЛ), сгруппированных по дате. Отметь матчи галочками и нажми «Сохранить выбор»:
- Выбранные матчи становятся `featured` и появляются на главной странице
- Для каждого выбранного матча Бендер генерирует прогноз и делает свою ставку
- Предыдущие комментарии Бендера удаляются и создаются заново

Лимит: 10 запросов в минуту.

---

## Панель суперадмина (`/superadmin`)

Доступна только пользователям с флагом `is_superuser`.

### Симуляция результатов

Чеклист запланированных матчей. Кнопка «Симулировать выбранные»:
- Выбранные матчи получают статус `finished` и счёт 1:0
- Пересчитываются очки всех участников
- Бендер генерирует новый комментарий к таблице очков

Используется для тестирования. Лимит: 10 запросов в минуту.

### Матчи АПЛ (тестовый функционал)

Временная кнопка для загрузки матчей английской Премьер-лиги:
- Опциональная галочка «Удалить существующие матчи АПЛ перед загрузкой» — очищает ставки и результаты по АПЛ перед синхронизацией
- Матчи загружаются с football-data.org (соревнование PL)

### Опасная зона

| Действие | Подтверждение | Что удаляется |
|---|---|---|
| Сбросить ставки и очки | Одиночный confirm | Все прогнозы и PredictionPoints. Пользователи и матчи сохраняются. |
| Сбросить базу данных | Двойной confirm | Полный сброс: все таблицы пересоздаются, запускается начальный сид. |

## Лог активности (`/activity-log`)

Доступен только суперадмину. Ссылка «📋 Лог активности» в шапке суперадмин-панели.

**Фильтры:**
- Период «С» — «По» (по умолчанию: последние 7 дней)
- Конкретный пользователь или «Все»

**Таблица** (до 500 записей, новые сверху):

| Колонка | Описание |
|---|---|
| Дата/время | UTC-время события |
| Пользователь | Имя, заметка суперадмина в скобках, иконка роли |
| Действие | Цветная плашка с типом события |
| Детали | Текстовое описание (что именно сделано) |
| IP | Адрес клиента |

**Типы событий:**

| Код | Название |
|---|---|
| `login` | Вход в систему |
| `prediction_set` | Ставка |
| `password_changed` | Смена пароля |
| `profile_updated` | Обновление профиля |
| `admin_password_reset` | Сброс пароля пользователя |
| `admin_toggle` | Изменение прав |
| `results_simulated` | Симуляция матчей |
| `scores_reset` | Сброс ставок и очков |
| `db_reset` | Сброс базы данных |
| `theme_changed` | Смена темы |
| `featured_set` | Настройка матчей для ставок |
| `user_created` | Создание пользователя |
| `user_deleted` | Удаление пользователя |
| `pl_matches_loaded` | Загрузка матчей АПЛ |

## Управление пользователями

Все действия — в панели суперадмина (`/superadmin`).

### Создание пользователя

Поля: Логин (обязательно, минимум 2 символа), Пароль (минимум 3 символа), Никнейм (необязательно).

Логин проверяется на уникальность. После создания страница перезагружается.

### Список пользователей

Для каждого пользователя (кроме суперадмина и ботов) доступны:

- **Заметка суперадмина** — приватная метка до 100 символов, видна только суперадмину. Редактируется через кнопку ✏️, сохраняется инлайн.
- **Назначить / Снять** — выдать или отозвать права администратора.
- **Сброс пароля** — сбрасывает пароль пользователя до его логина (пароль = логин).
- **Удалить** — удаляет пользователя с двойным подтверждением. Удаляются также все его ставки и очки. Логи сохраняются (поле `user_id` обнуляется).

**Защиты при удалении:**
- Нельзя удалить суперадмина
- Нельзя удалить самого себя
- Нельзя удалить бота

## Темы оформления

Четыре цветовые темы, переключаются в суперадмин-панели:

| Тема | Акцентный цвет |
|---|---|
| Синяя (navy) | Синий |
| Зелёная (forest) | Зелёный |
| Фиолет (purple) | Фиолетовый |
| Алая (crimson) | Красный |

Тема сохраняется в базе данных (таблица `settings`, ключ `theme`) и применяется глобально для всех пользователей через атрибут `data-theme` на теге `<body>`.

Панели Бендера (золотая и зелёная) не зависят от темы — их цвета фиксированы.
